# سایت پیش‌سفارش شام مهمانی 🍽️

سایت کوچکی برای اینکه مهمان‌های مهمانی خداحافظی، شام خود (غذا، پیش‌غذا و نوشیدنی)
را از قبل انتخاب و ثبت کنند تا سفارش با نام هرکس به رستوران تحویل داده شود.

منو از منوی رستوران **باکس** (boxrestaurant.ir) استخراج شده است. قیمت‌ها نمایش داده **نمی‌شوند**.

## راه‌اندازی

```bash
npm install
npm start
```

سپس مرورگر را روی http://localhost:4123 باز کنید.

### تنظیمات (متغیرهای محیطی)

| متغیر | پیش‌فرض | توضیح |
|---|---|---|
| `PORT` | `4123` | پورت سرور |
| `ADMIN_PASSWORD` | `mehmooni` | رمز صفحه‌ی مدیریت — **حتماً تغییر دهید** |
| `PARTY_TITLE` | `مهمانی خداحافظی` | عنوان بالای سایت |
| `PARTY_SUBTITLE` | `سفارش شام خود را…` | زیرعنوان |

نمونه‌ی اجرای امن:

```bash
ADMIN_PASSWORD='یک-رمز-قوی' PARTY_TITLE='بدرود امیر' node server.js
```

## استفاده‌ی مهمان‌ها

۱. نام و نام خانوادگی را وارد می‌کنند.
۲. منو را می‌بینند و با دکمه‌ی «افزودن» آیتم‌ها را به سفارش اضافه می‌کنند (تعداد قابل تنظیم است).
۳. با دکمه‌ی پایین صفحه سفارش را مرور و «ثبت نهایی» می‌کنند.
۴. اگر نوشیدنی یا پیش‌غذا انتخاب نکرده باشند، هنگام ثبت یک **یادآوری ملایم** می‌بینند
   (اجباری نیست؛ می‌توانند نادیده بگیرند و ثبت کنند).
۵. هر زمان با همان نام برگردند، سفارش‌شان بارگذاری و قابل ویرایش است.

> نوار دسته‌بندی‌ها هنگام پیمایش به بالای صفحه می‌چسبد تا جابه‌جایی بین دسته‌ها آسان باشد.

## گزارش‌ها برای رستوران — `/admin`

آدرس http://localhost:4123/admin را باز کنید و رمز مدیریت را وارد کنید. دو گزارش:

- **جمع‌بندی اقلام:** مجموع تعداد هر غذا در کل سفارش‌ها (برای آشپزخانه).
- **سفارش هر مهمان:** فهرست سفارش هر نفر با نامش.

هر دو دکمه‌ی **چاپ** و **کپی متن** دارند تا به‌راحتی به رستوران بدهید.

## ذخیره‌سازی داده‌ها

سفارش‌ها در یک فایل واحد `data/orders.json` ذخیره می‌شوند:

- تمام نوشتن‌ها از یک صف تک‌پروسه‌ای عبور می‌کنند (بدون نوشتن هم‌زمان).
- هر ذخیره: نوشتن در فایل موقت → `fsync` → جایگزینی **اتمیک** (`rename`).
  در نتیجه فایل هرگز نیمه‌نوشته/خراب دیده نمی‌شود.
- نسخه‌های پشتیبان چرخشی در `data/backups/` نگه داشته می‌شوند.

> این معماری برای بار کم (حدود ۵۰ مهمان) کاملاً امن است. سرور باید یک **پروسه‌ی واحد**
> با دیسک ماندگار باشد (VPS یا رایانه‌ی خودتان) — نه محیط بدون‌سرور (serverless) که
> فایل در آن پاک می‌شود.

## استقرار روی سرور با Docker + Nginx و دامنه‌ی menu.vazgroup.ir

اپ داخل Docker روی `127.0.0.1:4123` اجرا می‌شود (فقط از خود سرور در دسترس است) و
**Nginx میزبان** آن را برای `menu.vazgroup.ir` پراکسی می‌کند و HTTPS را با certbot می‌گیرد.
این روش وقتی مناسب است که پورت‌های ۸۰/۴۴۳ سرور از قبل توسط Nginx اشغال شده‌اند.

### گام ۱ — DNS
در پنل دامنه‌ی `vazgroup.ir` یک رکورد **A** برای زیردامنه‌ی `menu` بسازید که به **IP سرور** اشاره کند:

```
menu.vazgroup.ir.   A   <IP-سرور-شما>
```

منتظر بمانید تا DNS منتشر شود (`dig menu.vazgroup.ir` باید IP سرور را نشان دهد).

### گام ۲ — اجرای اپ با Docker
```bash
git clone <آدرس-مخزن> goodbyparty && cd goodbyparty
cp .env.example .env
nano .env          # ADMIN_PASSWORD را به یک رمز قوی تغییر دهید
docker compose up -d --build
```

بررسی: `curl http://127.0.0.1:4123/api/menu` باید پاسخ دهد.

> اگر پورت `4123` روی سرور شما هم اشغال است، در `docker-compose.yml` (بخش `ports`)،
> `config.js` و فایل Nginx یک پورت آزاد دیگر بگذارید (مثلاً `4187`).

### گام ۳ — تنظیم Nginx میزبان
فایل آماده در `deploy/nginx/menu.vazgroup.ir` است:

```bash
sudo cp deploy/nginx/menu.vazgroup.ir /etc/nginx/sites-available/menu.vazgroup.ir
sudo ln -s /etc/nginx/sites-available/menu.vazgroup.ir /etc/nginx/sites-enabled/
sudo nginx -t && sudo systemctl reload nginx
sudo certbot --nginx -d menu.vazgroup.ir   # افزودن خودکار HTTPS
```

همین. سایت روی **https://menu.vazgroup.ir** و صفحه‌ی مدیریت روی **https://menu.vazgroup.ir/admin** بالا می‌آید.

### دستورهای مفید

```bash
docker compose logs -f          # مشاهده‌ی لاگ‌ها
docker compose ps               # وضعیت کانتینرها
docker compose restart app      # ری‌استارت اپ
docker compose down             # توقف
docker compose up -d --build    # اعمال تغییرات کد
```

### داده‌ها و پشتیبان
سفارش‌ها در `./data/orders.json` روی **دیسک میزبان** ذخیره می‌شوند (volume در compose)،
پس با ری‌استارت یا بازساخت کانتینر **پاک نمی‌شوند**. برای پشتیبان دستی کافی است همین
پوشه را کپی کنید:

```bash
cp data/orders.json ~/backup-orders-$(date +%F).json
```

## ساختار پروژه

```
server.js          سرور Express: فایل‌های ثابت + API + مدیریت
config.js          پورت و رمز مدیریت
data/menu.json     منوی استخراج‌شده (۱۵ دسته، ۱۰۷ آیتم)
data/orders.json   سفارش‌ها (هنگام اجرا ساخته می‌شود)
public/            رابط کاربری مهمان و صفحه‌ی مدیریت
```
