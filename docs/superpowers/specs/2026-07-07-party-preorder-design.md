# طراحی: سایت پیش‌سفارش شام مهمانی

**تاریخ:** ۱۳ تیر ۱۴۰۵ (2026-07-07)

## هدف
مهمان‌های مهمانی خداحافظی (~۵۰ نفر) پیش از رفتن به رستوران، غذا/پیش‌غذا/نوشیدنی خود را
انتخاب و ثبت کنند تا سفارش با نام هرکس به رستوران داده شود. **قیمت‌ها نمایش داده نمی‌شوند.**

## منبع داده
منوی رستوران باکس (`boxrestaurant.ir`) — یک SPA انگولار که منو را از API پلتفرم
`live-menu.ir` بارگذاری می‌کند. چون API پشت WAF بود و curl را رد می‌کرد، صفحه با
Chrome (puppeteer) رندر شد و پاسخِ `Menu/Config5` گرفته شد. فیلد `p` هر دسته یک رشته‌ی
JSON از محصولات است. دسته‌های «پیک» و «بسته‌بندی» حذف شدند. نتیجه: **۱۵ دسته، ۱۰۷ آیتم**
که در `data/menu.json` ذخیره شد (title, desc, pic). تصاویر از CDN رستوران لینک می‌شوند.

## معماری
یک پروسه‌ی **Node.js + Express**، بدون مرحله‌ی build و بدون دیتابیس.
- `server.js` — فایل‌های ثابت + API + گزارش مدیریت
- `data/menu.json` — منو (ثابت) · `data/orders.json` — سفارش‌ها (هنگام اجرا)
- `public/` — رابط مهمان و صفحه‌ی مدیریت

## ذخیره‌سازی امن (نگرانی اصلی کارفرما)
`orders.json` یک شیء کلید-شده با نامِ نرمال‌شده‌ی مهمان است. هر نوشتن از یک **صف
تک‌پروسه‌ای (mutex)** عبور می‌کند: نوشتن در فایل موقت → `fsync` → `rename` اتمیک.
در نتیجه خواننده هرگز فایل نیمه‌نوشته نمی‌بیند و هیچ نوشتنی گم نمی‌شود. پشتیبان چرخشی در
`data/backups/`. **آزمون فشار:** ۶۰ درخواست هم‌زمان → فایل سالم، بدون گم‌شدن نوشته.

## جریان مهمان (موبایل‌محور، RTL)
نام → منو (کارت آیتم با عکس/توضیح + دکمه‌ی افزودن و استپر تعداد) → شیت مرور (+ یادداشت
اختیاری برای آشپزخانه) → ثبت نهایی → تأیید. ورود مجدد با همان نام، سفارش قبلی را برای
ویرایش بارگذاری می‌کند (نام روی دستگاه هم ذخیره می‌شود).

## API
- `GET /api/menu` · `GET /api/order?name=` · `POST /api/order {name,items,note}`
  (اعتبارسنجی همه‌ی idها نسبت به منو، سقف تعداد ۲۰)
- `POST /api/admin/report {password}` — پشت رمز عبور

## گزارش‌های رستوران — `/admin` (با رمز)
۱. **جمع‌بندی اقلام:** مجموع تعداد هر غذا در همه‌ی سفارش‌ها، گروه‌بندی‌شده بر اساس دسته.
۲. **سفارش هر مهمان:** فهرست هر نفر با نامش و یادداشتش.
هر دو با دکمه‌ی چاپ و کپی متن.

## اصول طراحی فارسی
RTL کامل · فونت وزیرمتن (بسته‌بندی‌شده) · ارقام فارسی (۰–۹) · پالت گرم رستورانی ·
دکمه‌های بزرگ لمسی · حالت‌های بارگذاری/خالی/خطا به فارسی.

## امنیت
نام و یادداشتِ ورودی کاربر در سمت مهمان با `textContent` و در گزارش مدیریت با
`esc()` پاک‌سازی می‌شوند (جلوگیری از XSS). رمز مدیریت از متغیر محیطی.

## استقرار
باید روی یک پروسه‌ی واحد با دیسک ماندگار اجرا شود (VPS یا رایانه‌ی محلی) — نه serverless
(که فایل در آن پاک می‌شود). راه‌اندازی: `npm install && npm start`.
